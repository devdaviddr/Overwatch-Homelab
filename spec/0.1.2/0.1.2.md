# Overwatch Homelab — Product Specification v0.1.2

## Overview

v0.1.2 adds an in-app Help Center accessible from the sidebar. Users can browse markdown documentation covering platform basics, agent setup by operating system, and a FAQ including account management and troubleshooting. Content is rendered with full markdown support (code blocks, tables, blockquotes) styled to match the dark theme.

---

## Goals

- Give users a self-serve reference without leaving the app.
- Cover the most common setup friction points: agent configuration per OS and password reset.
- Render documentation in a clean, readable format consistent with the existing dark theme.

---

## UX Details & Acceptance Criteria

### 1) Sidebar Help Section (expandable)

- A **Help** item with a `HelpCircle` icon appears in the sidebar below the Resources section, separated by a divider.
- In **expanded** sidebar mode: clicking Help toggles an accordion that reveals the three topic links (Getting Started, Agent Setup, FAQ) indented below with a left border. A `ChevronRight` rotates 90° when open. The Help label is brand-coloured when any help route is active.
- Sub-links use `PlayCircle`, `Cpu`, and `HelpCircle` icons at `h-3.5 w-3.5`. Active sub-link is highlighted `text-brand-400 bg-brand-900/40`.
- The accordion **auto-expands** when navigating to any `/help/*` route (via `useLocation` effect).
- In **collapsed** sidebar mode: shows only the `HelpCircle` icon with a tooltip; clicking navigates directly to `/help/getting-started`.

Files changed:
- `apps/hub-client/src/components/Sidebar.tsx`

---

### 2) Help Page Layout

- Route: `/help/:topicId` — rendered inside `DashboardLayout` alongside the main sidebar.
- `/help` redirects to `/help/getting-started`.
- The page is **full-width** — topic navigation lives entirely in the main sidebar. `HelpPage` renders only the scrollable content area (`max-w-2xl` centered prose).

Files changed:
- `apps/hub-client/src/pages/HelpPage.tsx` (new)
- `apps/hub-client/src/App.tsx`

---

### 3) Markdown Rendering

- Uses `react-markdown` + `remark-gfm` (GitHub Flavored Markdown — tables, strikethrough, task lists).
- Custom component map applies Tailwind classes for every element to match the dark theme:
  - `h1` — white, 2xl, bottom border
  - `h2` / `h3` — white/gray-200, semibold with top margin
  - `p` — gray-300, relaxed line height
  - `code` inline — `bg-gray-800 text-brand-300`
  - `pre` + `code` block — `bg-gray-800 border-gray-700 rounded-lg`, horizontally scrollable
  - `table` — full-width, `border-gray-700`, alternating row tint
  - `blockquote` — brand-600 left border, italic gray-400
  - `a` — brand-400, opens in new tab

---

### 4) Help Content

Four markdown documents bundled via Vite `?raw` imports:

#### Getting Started (`getting-started.md`)
- What is Overwatch Homelab (key concepts table: Resource, Agent, Hub)
- Step-by-step: Creating your first resource
- Navigating Overview and Resource detail pages
- Live metrics explanation (heartbeat vs. metrics snapshot cadence)
- Note that metrics are not persisted in v0.1.x

#### Agent Setup (`agent-setup.md`)
- Prerequisites (Lab ID, Hub URL, Node.js 18+)
- **macOS** — why Docker is unsupported, wizard auto-launch, manual one-liner, launchd plist for persistent service
- **Linux** — one-liner, systemd unit file
- **Windows** — PowerShell one-liner, NSSM/Task Scheduler note
- **Docker** — compose and standalone invocation; macOS warning
- Environment variables reference table (`LAB_ID`, `HUB_URL`, `HEARTBEAT_INTERVAL_MS`, `METRICS_INTERVAL_MS`)
- Verification steps + connectivity troubleshooting

#### Architecture (`architecture.md`)
- Monorepo layout diagram (hub-server, hub-client, lab-agent, shared-types)
- Full ASCII component map showing all layers: agent → hub-server → PostgreSQL / hub-client
- Four annotated data flow diagrams:
  - Agent registration & heartbeat sequence
  - Dashboard subscription & live metrics relay
  - Authentication (login → JWT → API requests)
  - macOS agent auto-launch (capabilities check → spawn → pid tracking)
- Persistence model table (what lives where and for how long)
- Tech stack reference table (every major library with its role)

#### FAQ (`faq.md`)
- **Password reset** — Prisma Studio steps + `bcrypt.hash` node one-liner
- Changing display name / email via Prisma Studio
- Creating additional user accounts
- Agent offline — causes and fix steps
- Metrics not updating — checklist
- Docker/macOS VM specs issue
- Data persistence (v0.1.x is real-time only)
- What data the agent collects
- Deleting a resource
- Monitoring multiple machines
- Updating Hub URL / intervals post-creation
- `tsx` not found fix
- Hub port and `PORT` env var
- Rotating `JWT_SECRET`
- Reverse proxy config (nginx + Caddy)

## Rollback & Compatibility

- Entirely additive: no existing routes, components, or API endpoints are modified beyond adding a nav link and two new routes.
- Removing the feature requires deleting `HelpPage.tsx`, the `src/help/` directory, the two routes from `App.tsx`, and the Help block from `Sidebar.tsx`.
- `react-markdown` and `remark-gfm` can be uninstalled without affecting any other functionality.
